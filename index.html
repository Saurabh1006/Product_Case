Product Thinking Frameworks

Product Design Framework

1. Clarify
  • Target user?
  • User's goal?
  • Context of use?
  • Constraints?
2. Identify Users
  • User personas
  • Pain points
  • User journey
  • Jobs-to-be-done
3. Generate Solutions
  • Brainstorm features
  • Prioritize (impact/effort)
  • MVP definition
  • Wireframe key flows
4. Evaluate
  • Success metrics
  • Trade-offs
  • Risks
  • Next steps

Root Cause Analysis (5 Whys)

Process:
  • 1. State the problem clearly
  • 2. Ask "Why did this happen?" → Answer
  • 3. Ask "Why?" about that answer → Answer
  • 4. Repeat 5 times or until root cause found
  • 5. Identify actionable solutions

Prioritization Framework (RICE)

Reach

How many users affected per time period?

Impact

How much will it impact each user? (3=massive, 2=high, 1=medium, 0.5=low)

Confidence

How confident in estimates? (100%=high, 80%=medium, 50%=low)

Effort

Person-months required?

Score = (Reach × Impact × Confidence) / Effort

Behavioral Interview Framework (STAR)

Situation (20%)
  • Set the context briefly
  • When and where?
  • What was happening?
Task (20%)
  • Your specific responsibility
  • What needed to be achieved?
  • Why was it important?
Action (40%)
  • What YOU did (not "we")
  • Step-by-step approach
  • Why those decisions?
Result (20%)
  • Measurable outcomes
  • Impact on business/users
  • What you learned

Common Behavioral Themes to Prepare:

  • Leadership & Influence
  • Conflict Resolution
  • Failure & Learning
  • Data-Driven Decisions
  • Prioritization Under Constraints
  • Stakeholder Management

Metrics + Strategy Framework

Step 1: Understand the Goal
  • What business objective are we solving for?
  • Who are the key stakeholders?
  • What's the timeline?
Step 2: Define Metrics Hierarchy
  • North Star Metric: Single metric that captures core value
  • Primary Metrics: 3-5 metrics that drive the North Star
  • Secondary Metrics: Supporting/diagnostic metrics
  • Counter Metrics: Guard rails (quality, trust, safety)
Step 3: Build Strategy
  • How will we move these metrics?
  • What initiatives/features are needed?
  • Prioritize using RICE or similar framework
  • Define success criteria and timeline
